**ЧТО?**

Питон скрипт для проверки DNS резолверов. Скрипт качает через один DNS резолвер файл, замеряет скорость и сохраняет резолвер, если скорость вменяема. 

**ЗАЧЕМ?**

Есть ситуация, когда MasterDnsVPN при запуске считает домен рабочим, потому что DOWNLOAD_MTU = 3000, например. На практике оказывается, что резолвер либо шейпит скорость, либо вообще не дает себя эксплуатировать. Скриптом мы отсекаем такие резолверы, оставляя только годные.

**ЧТО НУЖНО?**

1. Python 3.14

2. curl

3. Клиент MasterDnsVPN + настроенный сервер

**КАК?**

1. Скачайте репозиторий и положите файлы в папку с MasterDnsVPN

2. Get some resolvers
    Для тестов брал отсюда: https://public-dns.info/nameserver/ru.html
    Если вы белосписочник: не ленитесь и сканируйте все резолверы, а не только операторские. Я двачевал через Ростелекомовские резолверы на МТС под БС, а к МТСовским коннекта не было.

3. Сохраняем резолверы в отдельный файл

4. Открываем tester_config.txt
    Если есть желание, редактируем под себя, я шукал резолверы с этими цифрами. Здесь я оставлю описание параметров. Внутри конфига писать ничего не стал, хз как нейрослоп это переварит.
   
        executable = MasterDnsVPN.exe # Тут указываем исполняемый для MasterDnsVPN
        resolvers = client_resolvers.txt # Тут живут наши резолверы, которые мы будем тестить
        dnsvpn_config = client_config.toml # Конфиг для MasterDnsVPN. Указывается руками на случай, если у вас заготовлено несколько конфигов.  
        link = https://packages.element.io/desktop/install/win32/x64/Element%20Setup.exe # Прямая ссылка на какой-нибудь файл. Лучше указывать большой по размеру. Можно оставить этот, он 180 мегабайт.
        start_port = 19000 # С этого порта начнут запускаться инстансы DnsVPN, игнорируя указанный порт в конфиге. Каждый последующий процесс увеличивает порт на +1.
        timeout_thread = 20 # Через сколько секунд скрипт гасит инстанс MasterDnsVPN и curl. Сделано на случай, если резолвер внезапно умер и DnsVPN начнет бесконечно его проверять. 
        timeout_curl = 10 # Время, отведенное curl на коннект + качку файла в секундах. Больше секунд - больше трафика тратится. Меньше секунд - выше шанс задушить процесс, не начав закачку => можем потерять потенциально рабочий резолвер.
        minspeed = 10000 # Отсеивание резолверов по скорости. Указывается в байтах. Имхо лучше оставить как есть и потом руками уже вытащить из csv файла высокоскоростные резолверы.
        threads = 10 # Количество потоков. Чем больше потоков, тем быстрее проверка. Большое число лучше не ставить, у меня начал троить сервер и из-за этого заведомо рабочие резолверы не проходили первоначальную проверку.

5. Запускаем скрипт 
   
   ```bash
   python3 dns_tester.py
   ```

6. По окончанию проверки открываем results.csv, копируем резолверы и пробуем выйти в интернет.
